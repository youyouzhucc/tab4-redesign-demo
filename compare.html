第四 Tab · 三方案讨论

结合 StockX Portfolio 等竞品，拆成 3 个可点方向，方便对齐「资产感 / 交易待办 / 服务工具」取舍。

方案 A · 资产盘

对标 StockX Portfolio

以持有估值为第一眼信息：总估值、今日涨跌、待鉴别/可挂售。默认落在「持有」，想要为辅；弱化还价。

  • 适合：强化「我的闲置值多少钱」、促挂售
  • 代价：还价/降价待办不在主路径
打开方案 A

方案 B · 交易台

现有好物方案强化 · 差异化优势

想要 / 还价 / 持有三态并排，强调待处理事务（降价、还价中、待回复）。鉴别通栏轻入口。

  • 适合：回访成交、议价转化（竞品少见）
  • 代价：资产金融感弱于 StockX
打开方案 B

方案 C · 服务台

对标得物服务 / Mercari Inbox

Tab 名「服务」：消息中心+待处理 + 鉴别类目列表。不做资产清单。

  • 适合：通知触达 + 鉴别能力入口
  • 代价：与「我的好物」资产心智弱绑定
打开方案 C
维度 A 资产盘 B 交易台 C 服务台
第一眼 持有估值 / 涨跌 想要·还价·持有数量 消息待办 + 鉴别
主任务 看值多少 → 挂售/鉴别 处理降价/还价 → 成交 处理通知 → 去鉴别
竞品影子 StockX Portfolio 自研组合（少见） 得物服务 / Inbox
讨论焦点 要不要做「资产感」 要不要把还价做进 Tab 第四 Tab 是资产还是工具

建议讨论顺序：先定第四 Tab 是「资产」还是「工具」→ 若资产，再选 A（估值）或 B（交易待办）。
